# CloudDrive – Relational Database Design
Relational database model for a Google Drive–like cloud storage system. The database manages users, files, and folders, including file metadata, versions, previews, permissions, comments, sharing, notifications, and activity tracking. It also supports hierarchical folder structures, user storage management, and special file states such as favorites, spam, and trash.

The project includes the design and documentation of the database architecture with:

Entity-Relationship (ER) diagrams

Relational schema diagrams

Table relationships and constraints

Normalization analysis

Foreign key and indexing definitions

These diagrams visually represent the structure of the database and the relationships between entities, providing a clear overview of system design and data organization.
